## 🚗 案例：估算汽车配置项对成交价和销量的贡献

### 🎯 一、业务目标

产品规划团队希望回答的问题是：

> 哪些配置是“消费者愿意为之付费”的？
> 哪些配置虽然看起来酷，但对销量没帮助？

换句话说，就是要量化每个配置项：

- 对**成交价（价格溢价能力）**的影响
- 对**销量（市场接受度）**的影响

最终输出可供产品经理使用的：

> 「配置收益贡献表」 + 「配置包优化建议」

---

### 📦 二、数据结构设计

#### 数据来源

来自厂商的车型明细表 + 实际成交数据（最好按月或季度）

| 字段               | 含义                                  |
| ------------------ | ------------------------------------- |
| `Model`            | 车型名称                              |
| `Trim`             | 配置版本（如豪华版、旗舰版）          |
| `Engine_Type`      | 动力形式（燃油 / 混动 / 纯电 / 增程） |
| `Battery_Capacity` | 电池容量                              |
| `ADAS_Level`       | 智驾等级（L2 / L2+ / L3）             |
| `Interior_Package` | 内饰风格（简约 / 豪华 / 运动）        |
| `Sunroof`          | 是否有天窗（0/1）                     |
| `Seat_Material`    | 座椅材质（织物 / 皮革）               |
| `Price`            | 成交均价（万元）                      |
| `Sales`            | 销量（台）                            |
| `Month`            | 销售月份                              |
| `Region`           | 销售区域                              |

> ⚙️ 提示：很多配置都是“虚拟变量（dummy variables）”，如 `Sunroof=1` 表示有天窗。

---

### 🧮 三、建模方法

我们可以分两个目标来建模：

#### 1️⃣ 模型一：配置对**成交价**的影响

目标：理解“价格形成机制”

模型形式（多元线性回归 / XGBoost 回归）：
[
Price = β_0 + β_1(Engine_Type) + β_2(Battery_Capacity) + β_3(ADAS_Level) + ... + ε
]

解释：

- 每个 (β_i) 表示“配置提升一单位（或从无到有）时，价格平均变化多少”
- 例如：

  - `ADAS_Level: +0.25` 表示智驾等级每提升一级，价格平均上涨 0.25 万元
  - `Leather_Seat: +0.8` 表示皮革座椅带来 8000 元溢价

工具：

- `sklearn.linear_model.LinearRegression`
- 或 `xgboost.XGBRegressor`（能处理非线性特征）

---

#### 2️⃣ 模型二：配置对**销量**的影响

目标：理解“消费者偏好强度”

模型形式（对数回归 / 分类模型 / XGBoost）
[
\log(Sales) = α_0 + α_1(Engine_Type) + α_2(ADAS_Level) + ... + ε
]

解释：

- 系数 (α_i > 0)：该配置与销量正相关（受欢迎）
- 系数 (α_i < 0)：该配置抑制销量（不被接受）
- 可计算**弹性系数**：配置变化 1 单位 → 销量变化的百分比

进阶：
你也可以把销量预测视为分类问题（热销 / 普通 / 冷门），
用 XGBoost 分类模型输出 feature importance。

---

### 🧭 四、结果解读与可视化

#### 1️⃣ 配置收益贡献表

| 配置项         | 对成交价贡献（万元） | 对销量影响（%） | 结论                   |
| -------------- | -------------------- | --------------- | ---------------------- |
| 智驾 L2+       | +0.6                 | +8%             | 明显受欢迎，建议标配   |
| 全景天窗       | +0.3                 | +2%             | 弱正向，可留在高配     |
| 豪华内饰包     | +0.7                 | -1%             | 提价多但不带动销量     |
| 电池容量+10kWh | +0.4                 | +10%            | 强拉动，建议优化续航   |
| 皮革座椅       | +0.8                 | +3%             | 稳定增益项             |
| 抬头显示 HUD   | +0.2                 | 0%              | 中性配置，可视成本优化 |

> 💡 这张表最能打动决策者：哪些配置“值得加钱”，哪些“消费者不在意”。

---

#### 2️⃣ 价格-销量响应曲线（Price Elasticity Curve）

通过模拟不同配置组合的价格 → 销量曲线，
可以得到每个配置包的“性价比最优点”。

比如：

```
X轴：售价（万元）
Y轴：销量预测
不同颜色：配置组合方案
```

👉 帮助回答：“旗舰版再加智能驾驶包，销量会不会掉太多？”

---

### 🧠 五、实际决策应用

| 应用方向             | 说明                                    |
| -------------------- | --------------------------------------- |
| **配置包优化**       | 保留高溢价+高销量配置，剔除低效配置     |
| **价格带设计**       | 确定入门、中配、旗舰三档的合理区间      |
| **产品定义会议支持** | 提供“数据支撑的配置优先级”              |
| **下一代车型规划**   | 预测未来主流配置组合（如高智驾+大电池） |

---

### 💬 六、总结一句话

> 通过回归或机器学习模型，我们可以把每个配置项转化为**“数据化货币”**：
> 看它到底为价格带来了多少溢价、为销量贡献了多少吸引力。
> 这让汽车产品规划从“经验决策”进化为“量化策略设计”。
